# Phase 4.5: UI/UX & Functional Refinement - Context

**Gathered:** 2026-05-11
**Status:** Ready for planning

<domain>
## Phase Boundary

This phase elevates the application's workspace usability and personalization. It focuses on sidebar flexibility (collapsible sections, dynamic icon rendering), filter interaction precision (click-to-deselect), and historical transparency (natural language activity logs with parent-task context). It also introduces user profile settings for time format display and resolves alignment issues in the layout shell.

</domain>

<decisions>
## Implementation Decisions

### Sidebar Workspace Controls
- **D-01: Section Toggles:** The "Projects" and "Focus Tags" sidebar sections will feature individual expand/collapse chevrons to allow users to hide/show these lists independently of the global sidebar state.
- **D-02: Deselection Logic:** Clicking an already selected Project or Tag filter in the sidebar will revert the active filter to "All Tasks" (or the default overview).

### Creation & Icon Rendering
- **D-03: Sidebar Create Actions:** A '+' (Plus) button will be added to the "Focus Tags" header in the sidebar to trigger the tag creation flow directly.
- **D-04: Dynamic Icon Rendering:** The sidebar will render the specific icon and color chosen by the user during Project/Tag creation, replacing static placeholder icons.

### History Readability & Context
- **D-05: Humanized Log Format:** Task updates will use a readable "Full Diff" format (e.g., "Updated description from 'X' to 'Y'") rather than technical property paths.
- **D-06: Subtask Context:** Activity history for subtask events (e.g., "Finished subtask X") will explicitly mention the parent task name (e.g., "Finished subtask X in parent task Y") to provide immediate context.

### Profile Settings & Visual Polish
- **D-07: Time Format Preference:** A new setting will be added to the Profile page allowing users to toggle between 12-hour (AM/PM) and 24-hour time formats.
- **D-08: Centered Sidebar Toggle:** The sidebar collapse icon (PanelLeftClose/Open) will be vertically and horizontally centered within the minimized sidebar track to match the navigation icon alignment.

### Claude's Discretion
- Exact animation timing for section toggles (default to 200ms spring).
- Layout of the "Time Format" setting in the Profile UI (ensure alignment with existing profile fields).
- Placement of the parent task name in audit logs (e.g., inline vs. tooltip).

</decisions>
