## Interaction Contracts

### 1. MainLayout Persistence
- `TopNav` and `Sidebar` MUST be rendered once in `App.tsx` and wrap the `<Outlet />`.
- Navigation between routes MUST NOT trigger a re-mount of the shell components.

### 2. Collapsible Sidebar
- **Expanded:** 280px wide. Shows Icon + Label.
- **Collapsed:** 72px wide. Shows Icon only.
- **Tooltips:** Collapsed state MUST show tooltips on hover for each menu item to provide context.
- **Transition:** 200ms ease-in-out for width and opacity of labels.
- **Trigger:** A chevron button at the bottom of the sidebar or in the TopNav.

### 3. Loading State
- Use a `Skeleton` wrapper for the Dashboard content during initial load.
- Avoid full-page spinners; keep the shell (Sidebar/Nav) interactive while content loads.

### 4. Component Refinement
- **Cards:** Reduce border-radius from `2rem` to `1rem` (16px) for a more professional look.
- **TopNav:** Remove "Operational Status: Optimal" and "Rise and Shine" greeting variants. Use a simple "Dashboard" or user name greeting.
- **Focal Point:** The Dashboard's primary focal point is the **Task List card grid**, which must be centered or dominantly placed to guide user attention immediately to their work items.
