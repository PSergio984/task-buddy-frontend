# Phase 4.2: High-Velocity Polish & Feature Hardening - Context

**Gathered:** 2026-05-11
**Status:** Ready for planning

<domain>
## Phase Boundary
This phase focuses on resolving functional gaps from 4.1 and refining the UX for high-velocity usage. It covers unified filter synchronization, the "Drawer-Only" interaction model for both creation and editing, and dashboard timeframe granularity.
</domain>

<decisions>
## Implementation Decisions

### 1. Unified Filter Synchronization
- **Decision**: Sidebar "Smart Lists" (Inbox, Today, Upcoming) will act as shortcuts that activate the corresponding filters in the Tasks page.
- **UX**: Clicking a sidebar link will navigate to `/tasks` and update the `FilterContext` so the Filtering Hub visually reflects the active state (e.g., the "Inbox" pill is highlighted).

### 2. The "Drawer-Only" Interaction Model
- **Decision**: Task creation will move from a centered Modal to a right-side Drawer, matching the viewing/editing experience.
- **Benefit**: Provides a consistent mental model where all "focused work" happens in the right-side workspace without blocking the main content visibility.

### 3. Drawer Workspace Optimization
- **Decision**: Remove the "Comments" section from the drawer.
- **Decision**: Expand the Description area to be the primary focus below the title.
- **Functional Subtasks**: Implement full CRUD for subtasks directly within the drawer (Inline add, toggle completion, and delete).
- **Performance**: Targeting transition duration for the drawer of 150ms-200ms.

### 4. Dashboard Granularity
- **Timeframe Switcher**: Add a global toggle (Today, Week, Month, All) to the Dashboard header to filter all status widgets simultaneously.
- **Widget Refinement**: Widgets will explicitly show their time-bound context (e.g., "Remaining Today").

### 5. High-Contrast & Safety Refinements
- **Sidebar Contrast**: Darker text/icons for better readability against the white background.
- **Filter Hub Contrast**: Higher contrast for panel backgrounds and active filter pills.
- **Safety**: Add a "Confirm Delete" dialog for tasks (using Radix UI).
- **Efficiency**: Implement a "Dirty Check" — only allow saving if changes have actually been made.

### 6. "Search and Create" Selectors
- **Pattern**: Update Project and Tag selectors to be searchable.
- **Inline Creation**: If a search doesn't match, users can press Enter or click a "Create '[name]'" button to instantly add the new project/tag without leaving the drawer.

### 7. Audit Trail Copy
- **Correction**: Change "Refined task '...': completed" to "Marked as done" or similar human-readable text.

</decisions>
