# Phase 03.8 Validation: Sidebar Organization & DND

## Overview
This document defines the validation strategy for Phase 03.8, ensuring that sidebar reordering and CRUD operations meet the specified requirements and maintain system integrity.

## Requirement Traceability

| Req ID | Requirement | Verification Method | Automated Command |
|--------|-------------|---------------------|-------------------|
| SIDE-01 | Reorder projects and tags via DND | E2E (Playwright) | `npx playwright test tests/sidebar-dnd.spec.ts` |
| SIDE-02 | Ellipsis menu for CRUD actions | Visual / Component Test | `npm run test src/components/sidebar/item-actions.test.tsx` |
| SIDE-03 | Edit/Delete project and tags | Integration Test | `npm run test src/components/sidebar.test.tsx` |
| SIDE-04 | Filter reset on deletion | Integration Test | `npm run test src/components/sidebar.test.tsx` |

## Verification Strategies

### 1. Drag-and-Drop Reliability
- **Focus**: Ensure `arrayMove` correctly updates the local order and `useReorder` mutations are triggered with correct ID sequences.
- **Tool**: Playwright.
- **Scenario**: Drag Project A below Project B. Verify the `reorder` API call contains `[B_id, A_id]`.

### 2. Interaction Model
- **Focus**: Ellipsis menu should not trigger sidebar selection when clicked.
- **Tool**: Vitest + React Testing Library.
- **Scenario**: Mock `onClick` for sidebar item and ensure it's not called when the menu trigger is clicked.

### 3. Filter Continuity (SIDE-04)
- **Focus**: Automatic fallback to "All Tasks" when the active context is deleted.
- **Tool**: Integration Test.
- **Scenario**: Set `activeSidebarFilter` to `project:123`. Call `deleteProject(123)`. Assert `activeSidebarFilter` is now `all`.

## Test Plan Execution

### Pre-execution
- Ensure backend reorder endpoints are functional (covered by backend tests).
- Verify `@dnd-kit` is available in `node_modules`.

### Post-execution
- Run full Playwright suite: `npm run test:e2e`
- Run sidebar unit tests: `npm run test src/components/sidebar/`
