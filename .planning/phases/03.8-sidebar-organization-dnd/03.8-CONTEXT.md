# Phase 3.8 Context: Sidebar Organization & DND

## Overview
This phase implements full CRUD capabilities for Projects and Tags directly from the sidebar and adds Drag-and-Drop (DND) functionality to reorder them. It also cleans up the sidebar interaction model to handle state transitions (like deletion) gracefully.

## Decisions

### 1. Drag-and-Drop (DND)
- **Library**: Use `@dnd-kit` (already installed).
- **Handle**: Use a dedicated `GripVertical` icon from `lucide-react` on the left of the item.
- **Constraints**: 
  - Dragging is restricted within sections (Projects cannot be dragged into Tags).
  - Persistence is immediate on drop using existing `useReorderProjects` and `useReorderTags` hooks.
- **Feedback**: Use standard sortable animations (transform/opacity) during drag.

### 2. CRUD Actions (Edit/Delete)
- **UI Pattern**: Use a "More" ellipsis (`MoreVertical`) dropdown menu for each item in the sidebar.
- **Placement**: Visible on hover for each Project/Tag item.
- **Modals**:
  - **Edit**: Reuse `CreateProjectModal` and `CreateTagModal` in edit mode (populated with existing data).
  - **Delete**: Use the existing `DeleteConfirmationModal`.

### 3. State Management & Side-Effects
- **Filter Reset**: If the user deletes a project or tag that is currently the active filter (i.e., `activeSidebarFilter` or `activeTagId` matches the deleted item), the application must automatically reset the filter to "All Tasks" (`activeSidebarFilter = "all"`, `activeTagId = null`).
- **Data Refresh**: Continue using TanStack Query invalidation (`invalidateQueries(["projects", "tags", "tasks"])`) to ensure all views update immediately after CRUD actions.

### 4. Component Structure
- **Sidebar**: Will be refactored to use `SortableContext` from `@dnd-kit/sortable`.
- **SidebarItem**: Extract a reusable `SortableSidebarItem` component to handle the grip, label, and ellipsis menu logic for both Projects and Tags.

## Reusable Assets
- `useProjects` / `useTags` hooks (already support reorder/update/delete).
- `DeleteConfirmationModal`.
- `CreateProjectModal` / `CreateTagModal` (props for `project`/`tag` editing already exist).

## Out of Scope
- DND for Tasks (strictly limited to Sidebar Projects/Tags for this phase).
- Nested projects/sub-projects.
