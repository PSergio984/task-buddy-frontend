# Phase 4.3: Task Interaction Precision & Information Density - Context

## Domain
Finalizing premium task interactions with full field support during creation, time-aware deadlines, visual color-coding for projects/tags, and high-fidelity audit trails.

## Decisions

### 1. Unified Creation & Edit Experience
- **Subtasks & Tags in Creation**: Supported. Maintains local state `pendingSubtasks` and `pendingTags`.
- **Search-and-Create**: Enter to create tags/projects in the combobox is a hard requirement.

### 2. Time-Aware Deadlines & Premium Time Picker
- **Custom Time Picker**: Instead of the browser's native `<input type="time">`, we will use a **premium Select-based picker** (HH:mm) that matches the `Plus Jakarta Sans` typography and the application's design system.
- **Font Consistency**: All inputs in the Date/Time picker will use standard `font-sans` (Plus Jakarta Sans) with appropriate weight (bold/medium), avoiding `font-black` where it looks jarring.

### 3. State Sync & Real-time Updates (Drawer)
- **Problem**: Drawer doesn't update when items are added/modified because it relies on a stale task snapshot.
- **Solution**: The `TaskDetailDrawer` will **refetch the task by ID** when opened in "view" mode using a dedicated query (e.g., `useTask(id)`). Subtask additions and tag detachments will trigger local cache invalidation to ensure immediate UI feedback.

### 4. Renaming "Labels" to "Tags"
- **Alignment**: The frontend will be updated to use "Tags" everywhere (sidebar, drawer, cards) to match the backend and user preference.

### 5. No Projects Policy
- **Decision**: Tasks are allowed to have no project.
- **Naming**: In the UI, these will be categorized under **"Inbox"** rather than "No Project" to feel more premium and organized.

### 6. Subtask Information Density
- **Pagination/Expansion**: For tasks with many subtasks, the drawer will initially show the first **5 subtasks**. A "Show [N] more subtasks" button will expand the list inline.

### 7. Colorized Toast Notifications
- **Consistency**: All toasts will use vibrant status colors (`success`, `destructive`, `warning`, `info`) based on their variant, ensuring high visibility and consistency with the premium UI.

### 8. Enhanced Audit Trail
- **Backend Specificity**: Subtask audit logs will include the **parent task name** (e.g., "Added subtask 'X' to task 'Y'").
- **Update Details**: "Updated task" logs will explicitly state what changed (e.g., "Updated 'Launch' priority from MEDIUM to HIGH").
- **Frontend Humanization**: `describeAction` in `audit-trail.tsx` will be updated to parse and display these richer details.
