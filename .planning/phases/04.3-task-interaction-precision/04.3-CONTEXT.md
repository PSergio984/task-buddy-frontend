# Phase 4.3: Task Interaction Precision & Information Density - Context

## Domain
Finalizing premium task interactions with full field support during creation, time-aware deadlines, and visual color-coding for projects/tags.

## Decisions

### 1. Unified Creation & Edit Experience
- **Subtasks & Tags in Creation**: The "New Task" mode in the drawer will now support adding subtasks and tags *immediately*.
- **Implementation**: We will maintain local state for `pendingSubtasks` and `pendingTags` in the drawer. When the task is created, these will be saved as a batch or sequentially.
- **Search-and-Create**: The "Enter to create" logic for tags and projects will be brought back and hardened.

### 2. Time-Aware Deadlines
- **Time Selection**: Add a time picker (using a simple select or a dedicated time input) to the due date section.
- **Validation**: Disable previous days in the `CalendarPicker`. If the selected day is today, disable past hours/minutes.

### 3. Visual Information Density
- **Color Coding**: Projects and Tags will support color selection from a predefined premium palette.
- **Icons**: Categories/Projects will have a set of selectable icons (e.g., Work, Personal, Shopping).
- **Sidebar**: The sidebar will reflect these colors and icons for quick visual identification.

### 4. UI Hardening & Cleanup
- **Redundant Actions**: Remove the "Edit" and "Delete" breadcrumbs/actions from the `TaskCard` since the drawer is now the single source of truth for these operations.
- **Delete Confirmation**: Investigate and fix the `AlertDialog` behavior to ensure it triggers correctly every time.
- **Contrast**: Increase contrast for the filtering bar and sidebar labels (darker text/borders) to ensure readability on white backgrounds.
- **Blue Gradient**: Apply the requested blue gradient accent to the top of the application layout.
- **Padding**: Adjust title padding in the drawer to ensure consistent spacing when adding/deleting items.

### 5. Activity Logs Context
- **Detailed Logs**: Update the `AuditTrail` humanization logic to include the specific task name for all create, update, and delete actions (e.g., "Deleted task 'Buy Milk'").
