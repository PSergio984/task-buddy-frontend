# Phase 4.3 Research: Precision UI & Feature Hardening

## Summary
Technical research for Phase 4.3 implementation, focusing on time selection, date validation, and visual information density.

## Findings

### 1. Time Selection & Date Validation
- **Component Strategy**: Since `shadcn/ui` lacks a native time picker, we will extend the `Popover` + `Calendar` component by adding a "Time Selector" footer.
  - UI: Two `Select` components (Hours, Minutes) in a flex row below the calendar.
  - State: Merge the `Date` from the calendar with the `time` selection using `setHours` and `setMinutes` from `date-fns`.
- **Validation**: 
  - Past dates: Use the `disabled` prop in `react-day-picker` (shadcn Calendar): `disabled={{ before: new Date() }}`.
  - Past times: If the selected date is "Today", disable hour/minute options that have already passed in the current local time.

### 2. Advanced Searchable Selectors (Enter to Create)
- **Pattern**: Use the `Command` component from shadcn inside a `Popover`.
- **"Create" Logic**: 
  - Maintain a `searchQuery` state.
  - If `searchQuery.trim()` doesn't match any existing tag/project, show a `CommandItem` with a `Plus` icon: `Create "${searchQuery}"`.
  - On select (Enter), trigger the creation mutation and immediately attach the new item.

### 3. Visual Information Density & Styling
- **Color Coding**: 
  - Extend Tag/Project models to include a `color` hex field (already exists in some models, needs UI implementation).
  - Use a predefined premium palette: `Slate`, `Red`, `Amber`, `Emerald`, `Blue`, `Indigo`, `Violet`.
- **Icons**: 
  - Use `Lucide` icons for Categories/Projects (e.g., `Home`, `Briefcase`, `ShoppingCart`).
- **Styles**:
  - **Top Accent**: `linear-gradient(135deg, #1e3a8a 0%, #3b82f6 100%)`.
  - **Filter Contrast**: Add `border border-slate-200/10` and `bg-slate-50/5` (dark mode) or `bg-slate-100` (light mode) to the filtering bar to distinguish it from the main content.

### 4. Audit Trail Context
- **Persistence**: To ensure delete actions record the task name, we must pass the task title to the `mutate` call or capture it in the `onSuccess` callback context of `useDeleteTask`.
- **Humanization**: Update `audit-trail.tsx` to handle `metadata.task_title` if provided in the audit log from the backend.

### 5. UI Simplification (Redundancy)
- **TaskCard**: Remove the floating "Edit" and "Delete" icons. The whole card is the trigger; secondary actions clutter the "Clean" aesthetic.
- **Drawer**: Add `padding-bottom` to the title section to ensure space for subtask/tag additions doesn't feel cramped.
