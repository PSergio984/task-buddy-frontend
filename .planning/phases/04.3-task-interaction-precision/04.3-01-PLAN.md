# Phase 4.3 Plan: Task Interaction Precision & Information Density (Refined)

## 🎯 Success Criteria
- [ ] Drawer updates subtasks/tags/projects in real-time without closing/reopening.
- [ ] Premium custom time picker (HH:mm) with consistent `Plus Jakarta Sans` typography.
- [ ] "Labels" renamed to "Tags" across the entire frontend.
- [ ] "No Project" labeled as "Inbox" in the UI for a premium feel.
- [ ] Subtasks list supports "Show more" expansion (initial 5 visible).
- [ ] Toast notifications color-coded by status (`success`, `destructive`, `warning`, `info`).
- [ ] Audit trail includes parent task info for subtasks and specific field change details.
- [ ] Search-and-Create flow for Tags and Projects is smooth (Enter to create).

## 📋 Implementation Tasks

### 1. [Logic] Drawer State Sync & Reactivity
- Update `TaskDetailDrawer` to refetch the task details on open if in "view" mode.
- Ensure successful mutations (add subtask, delete subtask, add/remove tag) trigger a local state refresh or cache invalidation.
- Fix: Tags and Projects added via the combobox should reflect immediately in the Drawer's main view.

### 2. [Component] Premium Custom Time Picker
- Replace the native `<input type="time">` with a custom Select-based picker.
- Use `Plus Jakarta Sans` with appropriate weights (no `font-black`).
- Design: Two select wheels (or dropdowns) for Hours and Minutes, matching the system's aesthetic.

### 3. [UX] Renaming & "Inbox" Labeling
- Perform a global rename of "Labels" to "Tags" in all frontend components and UI text.
- Update project selection and display to show "Inbox" when `projectId` is null/empty.

### 4. [UX] Subtask Expansion (Load More)
- Modify the subtask list in `TaskDetailDrawer` to show only the first 5 subtasks by default.
- Add a "Show [N] more subtasks" button that expands the list inline with a smooth animation.

### 5. [UI] Colorized Status-based Toasts
- Update `src/components/ui/toast.tsx` to ensure all status types have distinct, vibrant background/border colors.
- Audit all `toast()` calls to use correct `variant` (`success`, `destructive`, etc.).

### 6. [Logic] Enhanced Audit Trail (Backend & Frontend)
- **Backend**: Update `log_action` calls for subtasks to include `parent_task_name` in details.
- **Backend**: Ensure "Updated task" logs include field-level diffs (e.g., "Priority from LOW to HIGH").
- **Frontend**: Refactor `describeAction` in `audit-trail.tsx` to parse these richer details and render them clearly.

### 7. [Feature] Color & Icon Support Finalization
- Verify that Tags and Projects correctly display their color/icon on `TaskCard` and in the Drawer.
- Ensure the "Search-and-Create" flow (Task 9 in previous plan) is fully functional with these visual properties.

### 8. [Cleanup] UI Polishing
- Repair `Delete Confirmation` AlertDialog trigger.
- Remove redundant Edit/Delete actions from `TaskCard`.
- Finalize Drawer title padding and rhythm.

## 🚀 Execution Strategy
1. **Infrastructure**: Renames and Toast styles.
2. **Componentry**: Premium Time Picker and Subtask Expansion.
3. **Logic**: Drawer Reactivity and Refetching.
4. **Data**: Backend Audit Trail improvements.
5. **UI Audit**: Final check of Colors, Icons, and "Inbox" labeling.
