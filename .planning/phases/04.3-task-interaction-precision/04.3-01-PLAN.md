# Phase 4.3 Plan: Task Interaction Precision & Information Density

## 🎯 Success Criteria
- [ ] Subtasks and Tags can be added during task creation in the Drawer.
- [ ] Due Date picker includes time selection and prevents past selections.
- [ ] Delete confirmation works reliably with a clear UI.
- [ ] No redundant "Edit/Delete" actions on TaskCards or Drawer breadcrumbs.
- [ ] Tags and Projects have visual color/icon indicators.
- [ ] Search-and-Create flow for Tags and Projects is smooth (Enter to create).
- [ ] Audit logs specify the task name for all actions.
- [ ] Dashboard features a premium blue gradient accent and high-contrast filtering.

## 📋 Implementation Tasks

### 1. [Logic] Client-side Pending State for Subtasks & Tags (Drawer)
- Update `TaskDetailDrawer` to maintain `pendingSubtasks` and `pendingTags` states when `isCreateMode` is true.
- Ensure these are included in the final `createTask` payload.
- Fix the visibility of these sections in "New Task" mode.

### 2. [Component] Time Selection in Due Date Picker
- Integrate time selection (HH:mm) into the `DatePicker` component.
- Use a simple layout (e.g., side-by-side or below the calendar) for time picking.
- Ensure the time is saved into the task's `dueDate` ISO string.

### 3. [UX] Date/Time Validation (Disable Past)
- Configure `react-day-picker` to disable dates before today.
- If today is selected, disable times in the past (client-side validation).

### 4. [Fix] Repair Delete Confirmation AlertDialog
- Debug the `AlertDialog` trigger logic in `TaskDetailDrawer`.
- Ensure the "Delete" action correctly closes the drawer and updates the UI after success.

### 5. [Cleanup] Remove Redundant Edit/Delete Actions
- Remove "Edit" and "Delete" buttons from `TaskCard` (since click-anywhere opens the drawer).
- Remove redundant task actions from the Drawer breadcrumbs/header if they duplicate the footer/main actions.

### 6. [Feature] Color & Icon Support for Tags
- Add a `color` field and optional `icon` to the Tag model/schema.
- Implement a color preset picker in the Tag creation/edit UI.
- Display the color/icon on the `TaskCard` and in the Drawer.

### 7. [Feature] Color & Icon Support for Projects
- Add `color` and `icon` support for Projects/Categories.
- Update the Project selection UI to show these visual cues.

### 8. [UI] Drawer Title Padding & Rhythm
- Increase vertical padding and adjust spacing around the Drawer title.
- Ensure a consistent vertical rhythm when adding or deleting tasks (avoid layout jumps).

### 9. [UX] Search-and-Create Flow for Tags/Projects
- Refactor the Combobox to show a "Create '[search]'" button when no results match.
- Enable the "Enter" key to trigger the creation of a new Tag/Project from the search input.

### 10. [Logic] Task Names in Audit Trail Logs
- Update `describeAction` in `audit-trail.tsx` to extract and display the task name from the event details.
- Ensure "Deleted task 'X'" and "Updated task 'Y'" are explicitly humanized.

### 11. [Design] Premium Blue Gradient & High-Contrast Filtering UI
- Add a subtle blue gradient accent to the top of the main layout or dashboard.
- Improve the contrast of the filter hub (replace grey-on-white with higher contrast borders/text).
