# Phase 04.7: Notification System Integration - UI Specification

## 1. Notification Bell & Badge

### Visuals
- **Icon:** `Bell` from `lucide-react`.
- **Location:** `TopNav`, next to the Theme Toggle or User Profile.
- **Badge:**
  - **Color:** `destructive` (Red) or `accent` (e.g., Violet/Indigo from the brand palette).
  - **Shape:** Circular, small (ping animation if count > 0 is optional but preferred for "premium" feel).
  - **Typography:** Very small (e.g., `text-[10px]`), bold, white text.
  - **Position:** Top-right corner of the bell icon, slightly overlapping.
  - **Condition:** Only visible when `unreadCount > 0`. If `unreadCount > 9`, show `9+`.

### Interaction
- **Hover:** Slight scale-up and subtle background highlight.
- **Click:** Toggle the `Popover`.

---

## 2. Notification Popover (Dropdown)

### Container
- **Component:** Radix UI `Popover` (Shadcn `Popover`).
- **Width:** Fixed width (e.g., `w-80` or `w-96`).
- **Max Height:** `max-h-[500px]` with internal scroll.
- **Styling:** `backdrop-blur-3xl`, `bg-background/98`, `border-primary/10`, `shadow-2xl`, `rounded-[2rem]` to match the user dropdown.

### Header
- **Content:** Title "Notifications" (Bold) and "Mark all as read" button.
- **Style:** `p-4 border-b border-border/50 flex justify-between items-center`.
- **Button:** Small, ghost/link style, primary color.

### Notification Item
- **Layout:** Flex row with an icon, content (title/message), and timestamp.
- **Icon:**
  - `REMINDER_BEFORE`: `Clock` (Blue)
  - `REMINDER_DUE`: `AlertCircle` (Amber)
  - `REMINDER_OVERDUE`: `AlertTriangle` (Red)
  - `SYSTEM`: `Info` (Indigo)
- **Title:** `text-sm font-bold`.
- **Message:** `text-xs text-muted-foreground line-clamp-2`.
- **Timestamp:** `text-[10px] text-muted-foreground/60` (e.g., "5m ago").
- **Unread Indicator:** A small dot (accent color) on the left of unread items.
- **Background:**
  - **Unread:** `bg-primary/5` or `bg-accent/5`.
  - **Read:** Transparent.
  - **Hover:** `hover:bg-muted/50 transition-colors`.

### Empty State
- **Content:** "No notifications" illustration or text.
- **Style:** Centered, `py-12`, `text-muted-foreground/40`.

---

## 3. Real-time Toasts

### Trigger
- Arrival of a new `REMINDER_DUE` or `REMINDER_OVERDUE` notification while the app is active.

### Content
- **Title:** The notification title.
- **Description:** The notification message.
- **Action:** A "View Task" button that navigates to the `action_url`.

---

## 4. Profile Settings (Push Toggle)

### Section
- **Location:** `ProfilePage`, under a new "Preferences" or "Notifications" card.
- **Components:**
  - `Label`: "Push Notifications"
  - `Switch`: Toggle for enabling/disabling.
  - `Description`: "Receive browser notifications for upcoming and overdue tasks."

### Logic
- **Enable:** Triggers `Notification.requestPermission()`. If granted, subscribes and posts to backend.
- **Disable:** Unsubscribes (optional) or just updates backend to ignore.
- **Status Check:** On page load, check if `Notification.permission === 'granted'` and reflect in UI.
